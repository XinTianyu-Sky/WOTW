## 1. 游戏概述

### 1.1 一句话简介（Elevator Pitch）

**《风尘叹》** 是一款以中国传统武侠江湖为背景的2D开放世界角色扮演游戏。玩家将扮演一名身世成谜的江湖游子，在风云变幻的虚构江湖世界中自由探索、拜师学艺、行侠仗义，最终揭开自己的身世之谜，并在这乱世中走出属于自己的江湖之路。

### 1.2 核心愿景（Vision）

打造一个**有温度的江湖**。不是冷冰冰的任务清单，而是一个会呼吸、有记忆、会对玩家行为做出真实反馈的鲜活武侠世界。每一个NPC都有自己的故事和人生，每一次选择都会留下痕迹，每一场战斗都关乎生死荣辱。

### 1.3 设计支柱（Core Pillars）

| 支柱 | 说明 |
|------|------|
| **自由探索** | 非线性的开放世界，玩家可以按照自己的节奏和方向游历江湖，没有强制路线 |
| **深度武学** | 多流派的武术体系，内外兼修，经脉贯通，每一套武功都有独特的招式和心法 |
| **江湖人情** | NPC具有复杂的关系网和记忆系统，玩家的行为会影响声望、人脉和江湖格局 |
| **养成策略** | 角色成长具有多维度选择，属性、武功、装备、经脉的搭配形成丰富的BD（Build）策略 |

### 1.4 目标用户画像

| 维度 | 描述 |
|------|------|
| **核心用户** | 25-40岁中国男性，有武侠小说/影视/游戏消费经历，偏好策略性和叙事深度 |
| **次级用户** | 18-25岁年轻玩家，通过手游接触武侠题材，偏好轻度上手但有深度可挖 |
| **情怀用户** | 35-50岁老玩家，怀念《金庸群侠传》《武林群侠传》等经典，追求怀旧体验 |
| **泛用户** | 对开放世界和角色养成有兴趣的泛RPG玩家 |

### 1.5 平台策略

| 阶段 | 平台 | 说明 |
|------|------|------|
| **首发** | iOS + Android（TapTap/App Store/各安卓渠道） | 移动端为主战场，免费下载+内购模式 |
| **第二阶段** | Steam（Windows/macOS） | PC端买断制，与移动端数据互通或独立发售 |
| **第三阶段** | Nintendo Switch（可选） | 若取得商业成功，考虑移植 |

移动端优先的原因：
- 中国武侠手游市场规模大（《烟雨江湖》月流水数千万级别）
- 独立开发者资金有限，移动端验证市场风险更小
- 2D像素/水墨风格天然适合移动端性能要求
- TapTap等社区提供零成本的用户获取渠道

## 2. 世界观设定

### 2.1 时代背景

故事发生在一个架空的古代世界——**大虞王朝**末年。

大虞立国三百载，皇室衰微，宦官专权，地方节度使拥兵自重。江湖势力趁势崛起，六大门派与三大世家明争暗斗，更有神秘组织「暗月楼」在暗中推波助澜。天灾频发，民不聊生，整个天下如同一座即将喷发的火山。

与此同时，江湖上流传着一个古老的预言：**「七星归位，天命易主」**。传说上古时期，有七件神器散落天下，当七星连珠之日，集齐七神器者将获得改天换命之力。这个传说，正悄然将整个江湖卷入一场空前的风暴之中。

## 3. 核心玩法循环

### 3.1 三层循环结构

```
┌─────────────────────────────────────────────────────┐
│                   【宏观循环：月】                      │
│  推进主线剧情 → 解锁新区域/门派 → 提升声望等级           │
│  → 参与武林大会/江湖大事件 → 改变世界格局               │
└──────────────┬──────────────────────────────────────┘
               │
    ┌──────────┴──────────┐
    │  【中观循环：天】     │
    │  接取支线任务         │
    │  → 修炼/练功          │
    │  → 探索地牢/秘境       │
    │  → 与NPC互动/送礼      │
    │  → 收集资源/制造装备    │
    └──────────┬──────────┘
               │
        ┌──────┴──────┐
        │【微观循环：分钟】│
        │ 战斗/对话     │
        │ → 获得奖励    │
        │ → 属性提升    │
        │ → 继续探索    │
        └─────────────┘
```

### 3.2 分钟级玩法（Moment-to-Moment）

玩家在屏幕上的操作：
- **移动探索**：点击/摇杆控制角色在2D俯视角地图上移动，与可交互物体（NPC、宝箱、采集点、机关）互动
- **战斗决策**：进入战斗后，每回合选择普通攻击、释放招式、使用道具、防御、逃跑。每回合限时30秒决策（可设置关闭）
- **轻功操作**：在特定地形（悬崖、河流、屋顶）使用轻功实现跨越，消耗内力
- **对话选择**：与NPC对话时出现选项，影响好感度、任务走向、获取信息
- **采集/互动**：点击草药、矿石、宝箱等进行采集

### 3.3 小时级玩法（Hour-to-Hour）

一次典型游戏会话（30-60分钟）的体验弧线：

```
进入游戏（上次下线位置）
    │
    ├─→ 查看今日任务/目标（5分钟）
    │     - 主线任务下一步
    │     - 日常任务（押镖、悬赏、门派贡献）
    │     - 限时事件（某地出现稀有商人/秘境开启）
    │
    ├─→ 战斗与探索（15-30分钟）
    │     - 前往任务地点，沿途触发随机事件
    │     - 进入地牢/秘境，进行多场战斗
    │     - 挑战Boss
    │
    ├─→ 城镇交互（10-15分钟）
    │     - 出售战利品
    │     - 与关键NPC对话推进关系
    │     - 在铁匠铺强化装备
    │     - 在药铺购买补给
    │     - 在客栈休息（存档/恢复）
    │
    └─→ 养成收尾（5分钟）
          - 分配修炼时间（离线修炼/在线修炼）
          - 检查武学突破条件
          - 整理背包
          - 存档/下线
```

### 3.4 天级玩法（Day-to-Day）

游戏内一天（现实约30-60分钟）的内容：

| 时段 | 推荐活动 | 机制影响 |
|------|----------|----------|
| **清晨（寅时-辰时）** | 练功、修炼内功 | 内功修炼效率+20% |
| **上午（巳时-午时）** | 探索、战斗 | 视野最佳，怪物活跃 |
| **下午（未时-申时）** | 跑商、交易 | 商人货物最全 |
| **傍晚（酉时）** | 与NPC社交 | 某些NPC只在此时出现 |
| **夜晚（戌时-丑时）** | 潜入、暗杀任务 | 潜行成功率+30%，但遭遇更强怪物 |

### 3.5 周级玩法（Weekly）

| 活动 | 频率 | 说明 |

### 4.2 武学系统

武学分为**外功**（招式）、**内功**（心法）、**轻功**（身法）三大类。每位角色同时只能**装备一套外功、一套内功、一套轻功**，但可以随时切换（非战斗状态）。

#### 外功（招式武学）

外功是战斗中实际使用的攻击技能。每套外功包含5-8个招式。

**外功设计范例：太极拳（武当派·柔系外功）**

## 5. 战斗系统

### 5.1 战斗框架

- **战斗模式**：回合制，我方与敌方交替行动
- **战场**：六边形网格战场（8×8格），支持地形高度差
- **出战人数**：玩家+最多3名同伴（4人队） vs 敌方1-8个单位
- **行动顺序**：根据所有单位的「速度」属性排序，每回合按序行动
- **操作模式**：点击选择招式 → 点击目标（单体）/ 确认范围（AOE） → 执行

**为什么选择六边形网格？**
1. 比四边形网格提供更自然的移动体验（六个方向 vs 四个方向）
2. 距离计算更精确，增强策略性
3. 在2D俯视角下表现良好，移动端点击操作友好
4. 区别于《烟雨江湖》的九宫格，形成差异化

#### 战场地形效果

| 地形类型 | 效果 |
|----------|------|
| **平地** | 无特殊效果 |
| **高地** | 对低地单位伤害+20%，被攻击伤害-10% |
| **水域** | 移动消耗+1格，火系伤害-30%，雷系伤害+50% |
| **草丛/竹林** | 站在其中的单位闪避率+15% |
| **火焰地面** | 每回合结束时受到灼烧伤害（最大生命值5%） |
| **冰面** | 移动时可能滑行额外1格（随机方向），雷系伤害+20% |
| **毒雾** | 每回合结束时叠加1层中毒 |
| **机关陷阱** | 触发后造成固定伤害或异常状态 |

### 5.2 内力系统（战斗核心资源）

- 战斗开始时，内力值=最大内力值
- 每回合开始时自动恢复「内力恢复」值（由定力+内功境界决定，通常3-8点）
- 释放招式消耗内力，奥义技消耗大量内力
- 内力归零后无法释放招式（只能普攻、防御、使用道具）

#### 内力属性（五行体系）

每种内功归属于一种五行属性，形成克制关系：

```
     火（克制金）
    ↗     ↘
  木       金（克制木）
  ↑         ↓
  水 ←——→ 土
（互克关系）

简化克制循环：金→木→土→水→火→金
```

| 五行 | 内功特性 | 代表内功 | 克制时伤害加成 |
|------|----------|----------|---------------|
| **金** | 锋锐、破甲 | 金刚不坏神功（少林） | +25% |
| **木** | 生长、恢复 | 长春功（百花谷） | +25% |
| **水** | 柔韧、控制 | 寒冰真气（雪山派） | +25% |
| **火** | 爆发、灼烧 | 九阳神功（少林）、圣火令心法（明教） | +25% |
| **土** | 厚重、防御 | 混元功（丐帮） | +25% |

当攻击方内功克制防御方内功时，最终伤害×1.25；被克制时×0.75。

### 5.3 招式系统

#### 招式类型

| 类型 | 特点 | 示例 |
|------|------|------|
| **普攻** | 无消耗，基础伤害 | 各外功的「起手式」 |
| **武技** | 消耗内力，中等伤害+效果 | 揽雀尾、寒冰掌 |
| **身法技** | 消耗内力，位移+攻击 | 燕返（向后跳1格，向前突刺2格） |

## 7. 成长体系

### 7.1 主线任务设计

主线故事分为**五章**：

| 章节 | 标题 | 时长 | 核心区域 | 解锁系统 |
|------|------|------|----------|----------|
| **第一章** | 风起于青萍之末 | 2-4小时 | 新手村→洛阳 | 基础战斗、装备、背包、任务 |
| **第二章** | 蜀道难 | 4-6小时 | 洛阳→蜀中 | 经脉、门派选择、同伴、制造 |
| **第三章** | 冰与火之歌 | 6-8小时 | 蜀中→北冥→西域 | 轻功升级、套装锻造、武林声望 |
| **第四章** | 东海潮生 | 5-7小时 | 中原→东海 | 飞行坐骑、终极武学、经脉全通 |
| **第五章** | 风尘叹（终章） | 6-10小时 | 全部区域 | 最终Boss战、多结局 |

**主线任务设计原则**：
- 每个任务都有多种完成方式（战斗/潜行/说服/金钱）
- 关键选择不可逆，永久改变世界状态
- 失败不等于Game Over，故事继续但走向不同
- 章节间有自由探索期，不强制立即推进

## 9. 技术选型建议

### 9.1 引擎对比

| 维度 | Unity | Godot 4.x | Cocos Creator |
|------|-------|-----------|---------------|
| **2D支持** | 良好 | 优秀（原生2D引擎） | 优秀（专注2D） |
| **移动端支持** | 优秀 | 良好 | 优秀（中国市场占有率高） |
| **跨平台** | 最佳 | 良好 | 良好（移动优先） |
| **学习曲线** | 中等（C#） | 较低（GDScript类Python） | 较低（TypeScript） |
| **社区资源** | 最丰富 | 快速增长中 | 中文资源丰富 |
| **性能** | 良好 | 良好（轻量） | 优秀（轻量，移动端优化佳） |
| **个人开发友好度** | 中等（引擎较重） | 优秀（~50MB，启动快） | 优秀 |
| **授权费用** | 个人免费（年收入<$200K） | 完全免费（MIT） | 免费 |
| **推荐度** | ★★★★☆ | ★★★★★ | ★★★☆☆ |

### 9.2 综合推荐：Godot 4.x

理由：
1. **个人开发者友好**：引擎轻量（~50MB），启动快，无需漫长的编译时间
2. **2D引擎原生**：真正的2D引擎，像素完美，不是3D改2D
3. **开源免费**：MIT协议，无授权顾虑，可深度定制
4. **跨平台**：一套代码发布iOS/Android/Windows/macOS/Linux
5. **GDScript**：语法简洁，学习成本低，适合快速迭代
6. **Tilemap系统**：非常适合制作俯视角2D RPG地图
7. **信号系统**：事件驱动架构天然适合复杂交互逻辑

## 10. 开发路线图

### 10.1 总体时间估算

> 以下为全职独立开发的估算。如有其他工作/学习，时间按比例延长。

| 阶段 | 时间 | 核心目标 |
|------|------|----------|
| **Pre-Production（前期准备）** | 1-2个月 | GDD定稿、技术验证、美术风格确定 |
| **MVP（最小可行原型）** | 3-4个月 | 核心循环可玩（移动+战斗+1个区域） |
| **Vertical Slice（垂直切片）** | 3-4个月 | 第一章完整可玩，含所有核心系统 |
| **Alpha（内容填充）** | 6-8个月 | 全部区域+全部系统+主线完成 |
| **Beta（打磨测试）** | 3-4个月 | 平衡性、Bug修复、性能优化 |
| **Release（发布）** | 1个月 | 商店页面、宣传、上线 |
| **Post-Launch（持续更新）** | 持续 | 新内容、活动、维护 |

**总计：约18-24个月到达可发布状态**

### 10.2 各阶段详细规划

#### Pre-Production（第1-2月）

| 任务 | 交付物 |
|------|--------|
| GDD定稿 | 本文档的最终版本 |
| 美术风格探索 | 3-5张概念图（角色、场景、UI风格） |
| 技术选型验证 | Godot 2D Tilemap大世界性能测试、战斗原型 |
| 核心系统原型 | 移动+对话+简单战斗的Godot原型项目 |
| 市场调研 | 竞品分析报告 |
| 项目仓库搭建 | Git仓库+CI/CD+项目管理 |

#### MVP阶段（第3-6月）

- 1个完整区域（洛阳及周边，含5-8个场景地图）
- 移动与交互（点击移动、NPC对话、采集、商店）
- 基础战斗（六边形网格、3个外功、基础AI）
- 角色系统（属性、装备、升级）
- 存档系统（本地存档+读档）
- 1个Boss（山贼头目，含完整Boss AI）
- UI框架（主菜单、HUD、背包、装备、对话UI）
- 昼夜系统（基础昼夜循环）

**MVP验证指标**：内部测试者30分钟内体验到探索→战斗→升级→换装备→打Boss的完整循环；千元安卓机稳定30FPS+

#### Vertical Slice阶段（第7-10月）

- 第一章完整剧情（主线+5条支线+出身系统）
- 门派系统（至少2个门派可加入）
- 完整战斗（内力、五行、连击、异常状态）
- 经脉系统、NPC系统（日程+好感度+记忆）
- 天气系统（全部天气类型）
- 同伴系统（2名同伴+AI战术）
- 制造系统（锻造+炼丹+烹饪基础版）
- 音频（区域BGM+战斗BGM+基础音效）

#### Alpha阶段（第11-18月）

- 全部六大区域、五章主线、50+条支线
- 全部门派（六大门派+三大世家）
- 8名同伴+个人故事线
- 30+套外功+20+套内功+10+种轻功
- 200+件装备+套装系统
- 20+Boss、完整制造系统、家园系统
- 跑商系统、成就系统（50+）
- 至少3个结局+隐藏真结局

#### Beta阶段（第19-22月）

- 平衡性调整（战斗数值、经济系统、掉落率）
- 性能优化（移动端帧率、加载时间、内存管理）
- Bug修复、封闭测试（100-200名核心玩家）
- 反馈迭代

#### Release阶段（第23-24月）

- 商店页面制作（TapTap/App Store/Steam）
- 宣传物料（预告片、截图、GIF）
- 社区运营（TapTap论坛、QQ群、Discord）
- 首发活动（折扣、限时活动、签到奖励）

### 10.3 风险与应对
